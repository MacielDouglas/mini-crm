# README Assets Guide

Este arquivo organiza os assets visuais usados no README do projeto.

## Estrutura recomendada

```text
docs/
├── README_ASSETS.md
├── landing-page.png
├── dashboard.png
├── pipeline.png
└── leads.png
```

## Screenshots que devem ser capturados

### 1. Landing Page

**Arquivo:** `landing-page.png`

**O que mostrar:**

- hero principal
- navbar
- CTA principal
- mockup ou seção visual mais forte

**Dica:**

- capture em resolução ampla
- prefira modo dark se estiver mais bonito visualmente
- tente centralizar a hero sem muito scroll

---

### 2. Dashboard

**Arquivo:** `dashboard.png`

**O que mostrar:**

- cards de KPI
- gráfico principal
- lista de leads recentes
- header autenticado

**Dica:**

- use dados preenchidos
- evite dashboard vazio
- capture sem zoom exagerado

---

### 3. Pipeline

**Arquivo:** `pipeline.png`

**O que mostrar:**

- board Kanban completo
- ao menos 3 ou 4 colunas
- leads distribuídos entre etapas

**Dica:**

- deixe alguns cards com valores e score visíveis
- essa é uma das telas mais importantes do projeto

---

### 4. Leads

**Arquivo:** `leads.png`

**O que mostrar:**

- listagem ou tela detalhada de lead
- filtros, busca ou timeline
- elementos que reforcem contexto comercial

**Dica:**

- capture uma tela que mostre que o sistema vai além do board

---

## Padrão visual recomendado

Para manter o README premium e consistente:

- use todas as screenshots com a mesma proporção
- prefira largura entre 1400px e 1800px
- comprima as imagens antes de subir
- mantenha o mesmo tema visual em todas as capturas
- evite dados sensíveis ou fictícios mal formatados

---
